# Roadmap

## Phase 1: Project Scaffold

- Create Fabric-style mod structure.
- Add Gradle and metadata placeholders.
- Add Java skeleton classes.
- Document physics and gameplay scope.

## Phase 2: Mod Bring-Up

- Update Minecraft, Fabric, Loom, mappings, and Java versions.
- Add or regenerate Gradle wrapper files.
- Confirm the mod loads in a development client.
- Register a minimal test item or block.

## Phase 3: Physics Core

- Implement physics object state updates.
- Add material definitions and lookup logic.
- Add gravity, damping, and basic collision response.
- Add unit tests or standalone debug checks where practical.

## Phase 4: Gameplay Prototype

- Spawn or display a physics object in-game.
- Map selected block/item types to physics materials.
- Tune behavior for readability and fun.
- Add debug controls or logs.

## Phase 5: Portfolio Polish

- Add screenshots or short demo media only after the prototype exists.
- Document limitations and tradeoffs.
- Write a concise engineering reflection on simulation design.
